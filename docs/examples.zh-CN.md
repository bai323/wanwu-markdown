# 真实使用示例

这些例子不是演示口号，而是这个工具当前适合做的几类小任务。

## 示例 1：保存一篇网页文章

目标：把一篇网页文章保存成 Markdown，方便放进 Obsidian。

步骤：

1. 打开文章页面。
2. 复制浏览器地址栏 URL。
3. 在工作台选择 `网页文章`。
4. 粘贴 URL，点击 `开始采集`。
5. 打开 `Markdown` 标签页检查结果。
6. 需要入库时，展开 `Obsidian 输出` 并点击 `存入 Obsidian`。

适合：公众号文章、普通博客、X 长帖。

不适合：需要复杂登录、反爬很强、或版权不允许保存的内容。

## 示例 2：整理一次 AI 对话

目标：把 Claude、ChatGPT、Gemini 或 Codex 的网页对话变成可读 Markdown。

步骤：

1. 打开目标对话。
2. 复制当前 URL。
3. 在工作台选择 `AI 对话`。
4. 粘贴 URL，点击 `采集 AI 对话`。
5. 如果页面权限导致采集失败，改用官方导出或手工复制后导入。

适合：个人学习记录、方案复盘、需求讨论。

注意：不要导出包含他人隐私、客户资料、账号密钥的对话。

## 示例 3：恢复 Sider 插件里的长对话

目标：不靠 URL，直接读取 Chrome 里 Sider 插件的当前对话。

步骤：

1. 在 Chrome 中打开 Sider 插件。
2. 打开你想保存的那段对话。
3. 回到万物 Markdown，选择 `浏览器插件`。
4. 选择正确的 Chrome 配置。
5. 点击 `重新检测当前对话`。
6. 选择目标对话，点击 `恢复所选对话`。
7. 查看 `分支`、`报告` 和 `Markdown`。

适合：很长、有分支、普通复制容易丢结构的对话。

## 示例 4：生成分支报告

目标：看清楚长对话里不同回复路径、分叉点和模型来源。

步骤：

1. 先完成一次对话采集或导入。
2. 打开 `分支` 标签页。
3. 查看左侧分叉列表和右侧对比列。
4. 点击 `生成分支报告`。
5. 在 `报告` 标签页预览 HTML 报告。

适合：比较多个模型回复、复盘提示词迭代、整理 agent 任务过程。

## 示例 5：导出训练数据草稿

目标：把对话整理成 JSONL 草稿，供后续标注或评测。

步骤：

1. 完成采集或导入。
2. 打开 `训练数据` 标签页。
3. 下载 JSONL。
4. 在真正使用前做去重、脱敏、质量筛选和人工抽检。

注意：这是草稿，不是最终训练集。公开或商用前需要明确授权和隐私处理。
